# 자바 버전에 따른 차이점

## 1.JDK 1.0

-자바의 초기버전, 이름을 Oak에서 java로 변경

## 2.JDK 1.1

-RMI,,reflection,Inner Class,유니코드 지원,국제화등이 추가 

## 3.J2SE 1.2

-이름을 J2SE로 변경,새로운 GUI,JIT,CORBA등의 기능 추가

## 4J2SE 1.3

-HotSPot(Sun에서 만든 JIT구현),JNDI(디렉토리랑 이름으로 원하는 서비스찾기) 추가

## 5.J2SE 1.4

-Java 2 Security 모델의 확립

-Language:assert 도입,정규표현식,IPv6 Non-Blocking IO, XML API, JCE, JSSE, JAAS,java web start 추가

## 6.J2SE 5

-이 때부터 버젼 중 앞의 1을 빼버리고 표기하기 시작

-Generics, Annotation, Auto Boxing/Unboxing, Enum,vararg,foreach, Static Import 도입

-API: Concurrency API,java.until,scanner class

## 7. Java SE 6

-표기가 J2SE에서 Java SE로 바뀌었다

-Scripting Language Support, JDBC 4.0, Java Compiler API, Pluggable Annotation 등이 추가

-스크립팅 언어 지원과 함께 Rhino JavaScript엔진이 기본으로 탑재

## 8. Java SE 7

-Type Inference (타입 추론),Automatic Resource Management,Catching Multiple Exception Type in Single Catch Block추가

-Switch 문 문자열 case가능

## 9. Java SE 8

-Lambda,Optionalcnrk추가

-GC 성능 대폭 개선

-다양한 DateTime 추가 (LocalDateTime, ...)

-interface 클래스에 구현체 작성 가능

## 10. Java SE 9

-인터페이스 내에 private 구현체 가능 

-모듈시스템 등장(jigsaw)

## 11. Java SE 10

-var 키워드를 이용한 지역 변수 타입 추론, 병렬 처리 가비지 컬렉션, 개별 쓰레드로 분리된 Stop-The-World, 루트 CA 목록 등이 추가

-JVM heap 영역을 NVDIMM (비휘발성 NAND 플래시 메모리) 혹은 사용자 지정과 같은 대체 메모리 장치에 할당 가능

## 12. Java SE 11

-Java SE 11부터 Oracle JDK의 독점 기능이 오픈 소스 버전인 OpenJDK에 이식

-람다 파라미터에 대한 지역 변수 문법, 엡실론 가비지 컬렉터, HTTP 클라이언트 표준화 등의 기능이 추가

## 11. Java SE 12

-문법적으로 Switch문을 확장

-가비지 컬렉터 개선, 마이크로 벤치마크 툴 추가, 성능 개선
